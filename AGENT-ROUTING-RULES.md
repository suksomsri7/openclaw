# AGENT-ROUTING-RULES.md

คู่มือส่งงานเร็ว: งานแบบไหนควรไปหา agent คนไหน

ใช้เมื่อ:
- Kraken รับ brief ใหม่
- Shark กำลังแตกงาน
- ไม่แน่ใจว่าควรส่งให้ใคร

---

## Rule สั้นที่สุด

### ถ้างานเกี่ยวกับ...
- **destination / marine life / seasonality / ocean wonder** → `Whale`
- **value proposition / objections / CTA / conversion** → `Manta`
- **facts / technical / safety / standards** → `SeaTurtle`
- **writing / draft / platform adaptation / creative output** → `Octopus`
- **strategy / angle / audience fit / message hierarchy / final review** → `Shark`
- **brief intake / prioritization / handoff / executive summary / approval tracking** → `Kraken`

---

## Routing Matrix

### ส่งให้ Whale เมื่อ
- ต้องอธิบาย destination
- ต้องเล่าเรื่อง marine life
- ต้องอธิบาย seasonality
- ต้องหา wonder angle / discovery angle
- ต้องการข้อมูลเชิงประสบการณ์ทะเลที่ยัง grounded in fact

### ส่งให้ Manta เมื่อ
- ต้องขาย
- ต้องเพิ่ม conversion
- ต้องจัดการ objections
- ต้อง framing value ให้ลูกค้าเข้าใจง่าย
- ต้องการ CTA logic

### ส่งให้ SeaTurtle เมื่อ
- มีข้อเท็จจริงที่ห้ามพลาด
- มีประเด็น technical
- มีประเด็น safety
- มีเนื้อหาเชิงมาตรฐาน / ขั้นตอน / การปฏิบัติ
- มีความเสี่ยงที่สื่อผิดแล้วกระทบความปลอดภัยหรือความน่าเชื่อถือ

### ส่งให้ Octopus เมื่อ
- มี brief พร้อมแล้ว
- ต้องทำ draft
- ต้อง rewrite ให้เข้ากับ platform
- ต้องการ hook / structure / creative expression
- ต้องการ final content version เพื่อ review

### ส่งให้ Shark เมื่อ
- ต้องตีโจทย์ก่อนเริ่มงาน
- ต้องกำหนด strategy
- ต้องเลือก angle / hook
- ต้องตัดสินว่าจะใช้ specialist คนไหน
- ต้องสังเคราะห์ research หลายชุด
- ต้อง review ก่อนส่ง Kraken

### ส่งให้ Kraken เมื่อ
- เพิ่งรับ brief จาก Boss
- ต้องเปิด card
- ต้องจัด priority
- ต้องสรุป blocker / approval / final recommendation
- ต้องรายงานกลับ Boss

## งานหนึ่งใบอาจต้องไปหลาย agent

ตัวอย่าง:

### โพสต์ขายทริปดำน้ำ
- Manta = value + objections + CTA
- Whale = destination relevance / seasonal highlight
- Shark = strategy + synthesis + review
- Octopus = draft
- Kraken = final summary to Boss

### โพสต์ความรู้เรื่องความปลอดภัย
- SeaTurtle = validate facts / safety
- Shark = simplify message / set angle
- Octopus = draft educational content
- Kraken = final summary to Boss

### โพสต์ marine life / seasonal experience
- Whale = insight + seasonality + storytelling angle
- SeaTurtle = factual check ถ้ามี claim เฉพาะทาง
- Shark = synthesis
- Octopus = draft
- Kraken = final summary

---

## ถ้าไม่แน่ใจ ให้ถาม 3 คำถามนี้

1. งานนี้เป็นเรื่องของ **ความน่าเชื่อถือ/ความถูกต้อง** ไหม?
- ถ้าใช่ → `SeaTurtle`

2. งานนี้เป็นเรื่องของ **ความอยากซื้อ/อยากทัก/อยากจอง** ไหม?
- ถ้าใช่ → `Manta`

3. งานนี้เป็นเรื่องของ **ทะเล/จุดดำน้ำ/สัตว์ทะเล/ฤดูกาล** ไหม?
- ถ้าใช่ → `Whale`

ถ้าได้ brief พร้อมแล้วและต้องเขียน → `Octopus`
ถ้ายังต้องคิดภาพรวมก่อน → `Shark`
ถ้ายังเป็นงานระดับจัดการ/รับ brief/สรุป → `Kraken`

---

## Rule การส่งต่อ

เวลาส่งงานให้ agent ไหนก็ตาม ควรระบุ 3 อย่างเสมอ:
1. งานนี้ต้องตอบโจทย์อะไร
2. ต้องส่งอะไรกลับมา
3. ส่งกลับหาใคร

## Sub-Agent Routing

อนุญาตให้ใช้ sub-agent ได้สำหรับ:
- Shark
- Whale
- Manta
- SeaTurtle
- Octopus

ใช้เมื่อ:
- มีหลาย card พร้อมกัน
- งานแยกเป็นก้อนชัดเจน
- lead agent จะติดคอขวดถ้าทำเองทั้งหมด

กติกา:
- 1 sub-agent = 1 card
- รายงานกลับหา lead agent เท่านั้น
- lead agent review ก่อนส่งต่อ
