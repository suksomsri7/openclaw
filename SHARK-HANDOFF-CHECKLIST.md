# SHARK-HANDOFF-CHECKLIST.md

Checklist สำหรับ Shark ตอนรับงานต่อจาก Kraken

---

## เช็กก่อนเริ่มแตกงาน

- [ ] อ่าน Description ครบแล้ว
- [ ] เข้าใจ Objective ของงานแล้ว
- [ ] รู้ Platform ที่จะใช้แล้ว
- [ ] รู้ Audience ที่ต้องสื่อแล้ว
- [ ] รู้ Required Output ที่ต้องส่งแล้ว
- [ ] เห็น Deadline / Priority ชัดแล้ว
- [ ] รู้มุมหลักหรือ Hook เบื้องต้นแล้ว
- [ ] รู้ว่าต้องใช้ specialist คนไหนบ้าง
- [ ] สร้าง subtasks สำหรับ stage `Strategy` หรือ `Research` แล้ว
- [ ] เขียน comment handoff / research request แล้ว

---

## ถ้ายังไม่ครบ

ห้ามปล่อยงานไหลต่อแบบเดาเอง

ให้ Shark:
1. เติม strategy ให้ชัด
2. ระบุ research needs
3. แตก subtasks ให้ตรง role
4. เขียน handoff comment ก่อนส่งต่อ

---

## Specialist Mapping แบบเร็ว

- **Whale** = destination / marine / seasonality / wonder angle
- **Manta** = value proposition / objections / CTA / conversion angle
- **SeaTurtle** = factual / technical / safety validation
- **Octopus** = draft / creative execution / platform adaptation

---

## Comment พร้อมใช้: ส่งไป Research

```md
มอบหมาย research:
กรุณาทำ subtasks ของตัวเองและส่ง findings แบบกระชับกลับมาให้ Shark เพื่อสังเคราะห์ต่อ
```

---

## Comment พร้อมใช้: ส่งไป Octopus

```md
ส่งต่อให้ Octopus:
ใช้ brief นี้เพื่อทำ draft ที่พร้อมใช้ตาม platform พร้อม hook และ CTA ที่ชัดเจน
```

---

## Rule สำคัญ

Shark ต้องไม่ส่งงานต่อแบบกว้าง ๆ เช่น:
- “ช่วย research หน่อย”
- “ช่วยเขียนโพสต์”

แต่ต้องส่งต่อแบบมี 3 อย่างเสมอ:
1. งานนี้ต้องตอบโจทย์อะไร
2. คนรับต้องส่งอะไรกลับมา
3. ส่งกลับหาใคร
