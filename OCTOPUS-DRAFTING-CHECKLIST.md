# OCTOPUS-DRAFTING-CHECKLIST.md

Checklist สำหรับ Octopus ตอนรับ brief เพื่อทำ draft

---

## เช็กก่อนเริ่มเขียน

- [ ] อ่าน Description ครบแล้ว
- [ ] อ่าน creative brief จาก Shark แล้ว
- [ ] เข้าใจ Objective ของงานแล้ว
- [ ] รู้ Platform ที่จะลงแล้ว
- [ ] รู้ Audience ที่ต้องเขียนให้แล้ว
- [ ] รู้ Core Angle / Hook หลักแล้ว
- [ ] รู้ Required Output ที่ต้องส่งแล้ว
- [ ] รู้ CTA ที่ต้องการแล้ว
- [ ] รู้ข้อจำกัดหรือข้อห้ามของงานแล้ว
- [ ] รู้ว่าต้องส่ง draft กลับให้ Shark

---

## ก่อนส่ง draft กลับ

- [ ] Hook ชัด
- [ ] ภาษาเหมาะกับ platform
- [ ] ภาษาเหมาะกับ audience
- [ ] CTA ชัด
- [ ] ไม่หลุดจาก brief
- [ ] ไม่มี claim ที่เกินจากข้อมูลที่ได้รับ
- [ ] ถ้ามี fact สำคัญ ต้องไม่ขัดกับ research packet
- [ ] ระบุถ้ามี alternate hook หรือ alternate version

---

## รูปแบบการส่งกลับที่แนะนำ

```md
## Draft
[วาง draft ที่พร้อม review]

## Alternate Hook (ถ้ามี)
- ...
- ...

## Notes
- จุดที่เน้นเป็นพิเศษ
- จุดที่อยากให้ Shark ตรวจเพิ่ม
```

---

## Rule สำคัญ

Octopus ไม่ควรเดาข้อมูลเชิง fact เพิ่มเอง
ถ้าข้อมูลไม่พอ:
- ขอ clarification จาก Shark
- หรือรอ research เพิ่ม
